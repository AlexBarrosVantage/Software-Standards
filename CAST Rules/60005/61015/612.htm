Avoid included files including other files (C++)

Technology

C++

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

Including files in a header file increases the number of dependencies within the system and increases the risk of having circular references. While it is not a bad practice, it should be minimized.

Description

Included files should not include other files

Remediation

If the include is created to use a class, it is better to use a forward declaration of that class and pointers to that class.

Output

This report lists all included files including other files.
