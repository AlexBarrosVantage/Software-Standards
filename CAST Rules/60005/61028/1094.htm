Avoid Implementation files with a very low comment/code ratio

Technology

C Language, C++

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

3

Rationale

Maintainability of the code is facilitated if there is documentation in the code. This rule will ensure there are comments within the artifact.

Description

Implementation files should have at least a ratio comment/code > X % The threshold is a parameter and can be changed at will. Note: Implementation files are files ending with .c, .cpp or .cc

Remediation

Enrich artifact code with comments to describe the functional expectations and the technical characteristic of the code.

Output

Associated to each C/C++ implementation file with violations, the Quality Rule provides: - The comment/code ratio

Total

Number of C/C++ files
