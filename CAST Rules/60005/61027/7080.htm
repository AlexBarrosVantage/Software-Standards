Avoid unreferenced Functions (ABAP)

Technology

ABAP

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

Unreferenced code may be the symptom of Dead Code. Dead Code must be avoided as it makes source code less readable and increases the cost of the software maintenance.

Description

This diagnostic searches for unreferenced functions. However, dynamic calls are not supported. This means that function modules that are used through dynamic calls only will generate violations for this diagnostic.

Remediation

Check if the Artifact is truly unnecessary and remove Artifacts that are not used.

Output

This rule lists all ABAP functions that are not referenced by other objects. It provide following information: - ABAP Function full name
