Rationale

All Functions, procedures and package functions & procedures should be referenced. Unreferenced code may be the symptom of Dead Code. Dead Code must be avoided as it makes source code less readable and increases the cost of the software maintenance. Warning: Unreference code can also be the symptoms of missing code (code not included in the source code analysis) and / or can be the symptom of use of polymorphism.

Description

List of schema functions, procedures and package functions & procedures that are not called A such objectt is a code that is not explicitly called from within the analyzed code.

Remediation

Check if the functions, procedures and package functions & procedures is truly unnecessary. Remove it if so

Output

This report lists all unreferenced unctions, procedures and package functions & procedures. It provides the following information: - Functions, procedures and package functions & procedures full name.

Total

total is the number of unctions, procedures and package functions & procedures stored into the schema
