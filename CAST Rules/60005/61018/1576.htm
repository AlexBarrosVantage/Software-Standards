Rationale

CHAR should be used for storing fix length character strings. String values will be space padded before stored on disk. If this type is used to store variable length strings, it will waste a lot of disk space. So, Oracle suggests to use VARCHAR2 instead of CHAR while declaring datatype. Also VARCHAR is going to be replaced by VARCHAR2 in future so Oracle recommends to replace VARCHAR by VARCHAR2.

Description

You use the VARCHAR2 datatype to store variable-length character data. The STRING and VARCHAR are subtypes of VARCHAR2 having the same range of values as their base type. For example, VARCHAR is just another name for VARCHAR2. VARCHAR is synonymous with VARCHAR2. However, in future releases of PL/SQL, to accommodate emerging SQL standards, VARCHAR might become a separate datatype with different comparison semantics. Therefore the recommendation from Oracle is to change char to varchar2 to save space and change varchar to varchar2 to accomodate future changes.

Remediation

Change CHAR columns to VARCHAR2. Change VARCHAR columns to VARCHAR2.
