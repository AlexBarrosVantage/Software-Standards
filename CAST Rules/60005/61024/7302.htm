Avoid using Pointers

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

Pointers are notoriously error-prone. They require a lot of vigilance from developers and they make the code difficult to maintain.

Description

This rule searches for Cobol programs containing data declarations for which the usage clause is POINTER.

Remediation

Avoid using pointers as much as possible.

Sample
05  EXAMPLE-P          USAGE IS POINTER.
Output

Associated to each COBOL Program with violations, the Quality Rule provides: - The number of violation patterns - Bookmarks for violation patterns found in the source code: - Data declaration with the usage set to POINTER

Total

Number of Cobol Programs
