Rationale

Essential Complexity equals the minimum number of tests to check the Artifact's non-structured behavior. High EC means more testing and higher risk of errors.

Description

Avoid Artifacts with High Essential Complexity (EC > 5). Complex functions are difficult to maintain. Keeping Artifacts small and simple ensures that code retains a good readability level. The threshold level is a parameter that can be changed at will.

Remediation

Review the design of the Artifact to reduce the number of non-structured independent paths. E.g.: Reduce the number of EXIT statements.
