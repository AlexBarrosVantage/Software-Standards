Avoid using ALTER

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

9

Rationale

The ALTER statement is error-prone. If it is used in Cobol programs, then GO TO statements as they appear in the listing may not be those that will be encountered by the program at run time. The ALTER statement makes the maintenance programmer's job more difficult.

Description

This rule searches for Cobol programs using ALTER statements to manage their control flow.

Remediation

If you need to change the processing sequence due to a certain condition, then use an alternative set of PERFORM or GO TO statements.

IBM Enterprise Cobol for z/OS - Programming Guide
