Avoid table and column names that are too long (portability)

Technology

JEE

Unified Status

N/A

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

If you specify table name too long you won't be able to create it on some database. This will prevent you to use these table name unless you use a Naming Strategy to reduce the length of the table name. To ease portability, choose table name with a length accepted by the majority of database you want to support.

Description

Reports all entities or column defined through annotation or XML that define a table or column name length greater than 30. The table and column name length are parameters that can be changed at will.

Remediation

Change the table name size or use a Naming Strategy to reduce the table name length. Some facts from http://www.mssqlcity.com/Articles/Compare table name length column name length SQL Server 2000 128 128 ASE 12.5 30 30 MYSQL 4.1 64 64 DB2 8.1 128 128 Oracle 9i 30 30

Sample
<class name="cast.core.InputFieldValueResource" table="cast_core_input_field_value_resource">
   ...
</class>
Remediation sample
<class name="cast.core.InputFieldValueResource" table="input_field_value_resource">
   ...
</class>
Output

This report lists all entities or property defined through annotation or XML that define a table or column name length greater than 30. It provides the following information: Persistent Entity name or Persistent Property name

Total

The total number of Persistent Entities and Properties
