Avoid Functions/Procedures with SQL statement including Subqueries (PL/SQL Oracle)

Technology

PL/SQL

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

9

Rationale

Having complex queries in artifacts can cause performance problems. Queries with Subqueries are considered complex.

Description

Functions / Procedures should not use SQL statement including Subqueries (at least subqueries should be avoided). Note: "Functions & Procedures" is a generic name that include different types of objects: - function - stored procedure - trigger (special stored procedure fired by event) - package function - package procedure

Remediation

Review the design of the query to avoid if possible Subqueries

Output

This report lists all Functions / Procedures with SQL statement including Subqueries. It provides the following information: Function / Procedure full name

1648
