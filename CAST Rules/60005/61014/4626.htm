Rationale

The keyword default is executed when none of the conditions being tested for in the switch statement are met or executed. Having no default, means that there is no backup. The cases that are "impossible" today are the ones most likely to be the causes of untraceable bugs in the future, when the impossible changes to the standard.

Description

Reports all methods that have no default statement in a switch block.

Remediation

Add the default statement with appropriate handling of unexpected values.

Output

This report lists all Methods with no default statement in a switch block. It provides the following information: - Method full name
