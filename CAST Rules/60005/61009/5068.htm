Avoid STOP RUN (use GOBACK instead)

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

STOP RUN stops the current process. In subprogram or in CICS programs, do not use STOP RUN but use GOBACK instead in order to return control to the calling process.

Description

This rule searches for Cobol programs using STOP RUN statements. Main batch programs (e.g: called by a JCL STEP) are not taken into account.

Remediation

Replace the STOP RUN statements by GOBACK in subprograms and in CICS programs.

Reference

IBM Enterprise Cobol for z/OS
