Rationale

In Oracle Application/eBusiness it is recommended not to use directly the call_form buit_in but "FND_FUNCTION.Execute" instead. The reason for that is to have a correct behaviour of the application. This rule also apply to other built-in like Open_form, clear_form etc. Outside of Oracle Applications/eBusiness context, encapsulating built-ins allow having a centralized control and ease upgrade of the application (for example when such a built-in is not supported anymore).

Description

"call_form" built-in Function should be used from a centralized location i.e. calls must be encapsulated. Thus the report list all usages of the call_form function. Legitimate calls should be marked as exceptions.

Remediation

Encapsulate the built-in and use the new function instead.

Reference

Oracle Applications recommendation on customization.

Sample
WHEN-BUTTON-PRESSED
     CALL_FORM('MODULE_B.FMX');
Remediation sample
WHEN-BUTTON-PRESSED
       MY_CALL_FORM('MODULE_B.FMX');

...

PROCEDURE MY_CALL_FORM( FORM_NAME VARCHAR2) IS
BEGIN
  -- Do Something
  CALL_FORM( FORM_NAME);
END;
