Avoid using literals in assignments (hardcoded values)

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

7

Rationale

In order to ease change and to avoid specific information potential misuse, it is highly recommended not to use literals (hard-coded values). Theu can be managed through constants.

Description

This metric retrieves all ABAP Artifacts using literals in assignments (hard-coded values). Statements that are taken into account are the following: - = assignment operator (ex: a = 'b'.) - Concatenate (ex: CONCATENATE a 'b' INTO c.) - Move (ex: MOVE 'a' TO b.) - Replace (ex: REPLACE 'a' WITH 'b' INTO c.) - Data initialization (ex: DATA a VALUE 'b'.)

Remediation

Try to use constant as much as possible in order to avoid using literals in the ABAP code.

Sample
wa_val = 'CD-15000'.
...
IF wa_val = 'CD-15000'.
  ADD 1 TO l_count1.
ENDIF.
Remediation sample
Remediation example:

CONSTANTS: rad     TYPE i VALUE 'CD-15000'.
...
wa_val = rad.
...
IF wa_val = rad.
  ADD 1 TO l_count1.
ENDIF.
