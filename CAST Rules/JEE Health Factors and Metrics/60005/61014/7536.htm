Avoid using AT Events in combination of LOOP AT .... WHERE constructs

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

7

Rationale

Processing level breaks in LOOP AT constructs depends on the content of internal tables. If these tables are sorted differently, then the expected logic of events can be modified. In addition, the AT events can be impacted by the WHERE condition.

Description

This metric retrieves all ABAP Artifacts which contain a the combination of a LOOP AT .... WHERE construct and an AT event such as : AT FIRST AT NEW AT END AT LAST

Remediation

Ensure that the internal table sorting order is fixed and that the event logic is compatible with the WHERE condition.

Reference

Enhancing the Quality of ABAP Development, Meijs-Krouwels-Heulmans-Sommen, SAP Press ISBN 1-59229-030-2 Page 102

Output

Associated to each ABAP Artifact with violations, the Quality Rule provides: - The number of violation patterns

Total

Number of ABAP Artifacts using LOOP AT .... WHERE constructs
