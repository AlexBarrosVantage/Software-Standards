Avoid declaring throwing an exception and not throwing it

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

When a method is declared to throw an exception but does not throw any exception: - the caller must write extra code to handle this potential exception - it makes the code difficult to read - the caller will have to write a try/catch block that will affect the performance of the application

Description

Reports all methods with code (i.e. not interface or abstract method or instance of generic method) that are declared to throw an exception that is never thrown in the method body. That is to say all methods: - declared to throw an exception in the signature - AND o not throwing this exception or subclass directly or via a method declared to throw it o OR that catch this exception Note that catching an exception is verified independently of where it occurs in the method code.

Remediation

Remove the throw declaration in the signature.

Reference

http://www.onjava.com/pub/a/onjava/2003/11/19/exceptions.html

Sample
public void aMethod() throws myException {
   System.println("a Method"); // VIOLATION
}
Remediation sample
public void aMethod() {
   System.println("a Method"); // FIXED
}
Output

This report lists all Methods declaring throwing an exception and not throwing it. It provides the following information: - Method full name - Exception Class Full Name

Total

number of methods declaring throwing an exception in the signature except methods from interface and abstract methods
