Rationale

Finally blocks must be used to execute the code that is needed after either the try and/or the catch block have been executed. It is usually the place to code the release of resources used in the try block. As such, an empty finally block is most probably the sign of potential "resource leaks" that will jeopardize the application's stability.

Description

This metric reports all methods with at least one empty finally block (empty or only containing comments). In a try and catch/finally statement, finally blocks should contain code to handle the thrown exception.

Sample
try { ... }
catch ( MyException e)
{ ,,, }
finally
{
  // DO NOTHING <= VIOLATION
}
Remediation sample
try { ,,, }
catch ( MyException e)
{ ,,, }
finally
{
  DoSomething();
}
