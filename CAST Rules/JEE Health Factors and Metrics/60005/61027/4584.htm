Rationale

Unreferenced code may be the symptom of Dead Code. Dead Code must be avoided as it makes source code less readable and increases the cost of the software maintenance. Warning: Unreferenced code can also be the symptoms of missing code (code not included in the source code analysis) and / or can be the symptom of use of polymorphism. The intensive use of polymorphism can make the application easier to change but much more difficult to test, debug and stabilize.

Description

All Classes should be referenced. An unreferenced Artifact is an Artifact that is not explicitly called from within the analyzed code. An unreferenced Class is a class 1- which is not referenced from other Java classes and 2- where none of its members is referenced from other java members of other classes. As of today, this excludes references from other languages (JSP calling static members - this case will be handled in the future).

Remediation

Check if the Artifact is truly unnecessary. Remove Artifacts that are not used.

Output

This report lists all unreferenced Classes It provides the following information: Class full name
