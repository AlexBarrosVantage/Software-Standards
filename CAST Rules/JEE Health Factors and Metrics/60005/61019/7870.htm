Avoid using the NOT LIKE operator in WHERE clauses

Technology

Weight in Parent Tech. Criterion

Rationale

A NOT LIKE operator in a WHERE clause does not allow to use database indexes and has an impact in terms of performance. It is better to invert the logical expression in data selection.

Description

This metric retrieves all ABAP artifacts containing Open SQL queries with NOT LIKE operator in their WHERE clause..

Remediation

If possible, avoid using the NOT operator in the WHERE clause because it is not supported by database indexes; invert the logical expression instead.
