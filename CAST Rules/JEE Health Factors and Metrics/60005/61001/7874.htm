Avoid using dynamic queries

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

7

Rationale

Dynamic queries are difficult to test and can decrease the code understanding. Moreover, include dynamic coding in UI components can generate security issues if the content of the dynamic clauses is not filter properly.

Description

This rule checks Abap artifacts that use dynamic Open SQL queries.

Remediation

Avoid using dynamic coding as much as possible and check if dynamic content is filtered properly.

Reference

NetWeaver - Secure Programming - ABAP http://help.sap.com/saphelp_nw2004s/helpdata/en/8f/35de1718944eb8a1462cf6362cc8b8/frameset.htm

Sample
select count(*) from (p_tab) into l_count.

select * from zcostfreq
where (where_tab).
endselect.
Output

Total

Number of ABAP Artifacts containing OpenSQL queries.
