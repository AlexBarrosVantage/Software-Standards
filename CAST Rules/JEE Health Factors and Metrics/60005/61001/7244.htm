Avoid direct usage of EJB Entity from the client

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

3

Rationale

An Entity Bean should always be used by the Bean Class of a Session Bean to reduce network call, to reduce coupling between the client and the implementation of business objects and also to avoid revealing your database structure to unknown application.

Description

Avoid direct usage of EJB Entity from the client. All calls to Entity Bean (implements javax.ejb.EntityBean) from the client (specified by the package name list) without passing through a Session Bean (implements javax.ejb.SessionBean) will be reported.

Remediation

Use an EJB Session as facade to encapsulate the complexity of interactions.

Reference

http://java.sun.com/blueprints/corej2eepatterns/Patterns/SessionFacade.html http://www-128.ibm.com/developerworks/java/library/j-ejb1008.html

Output

This report lists all Methods of the client that access directly to entity bean. It provides the following information: Method full name The client threshold can be changed at will.

Total

The ratio is the sum of client methods that call directly an Entity Bean divided by the total of direct and indirect call from the client to Entity Beans.
