Rationale

For maintainability and readability reasons, Classes should not have too many Constructors.

Description

Classes should have less than X Constructors. The threshold is a parameter and can be changed at will.

Remediation

Review the Class design to reduce the number of Constructors

This report lists all Classes with more than X Constructors. It provides the following information: Class full name, number of Constructors
