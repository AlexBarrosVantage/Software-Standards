Avoid using late binding

Technology

Rationale

Even if late binding offer advantages (like being version independent), it suffers from several disadvantages compared to early binding. With early binding: - Code is much faster - Compiler can detect error at compilation time rather than execution time - Intellisense is available

Description

Avoid using late binding. Late binding is done when a call to CreateObject is done AND that the variables are of type Object/Variant. Whenever a variable is typed during the definition (dim var as MyObject) it becomes early binding.

Remediation

Use early binding.

Sample
dim xl  as object
Set xl = CreateObject("Excel.Application")
Remediation sample
dim xl  as Excel.Application
Set xl = CreateObject("Excel.Application")
