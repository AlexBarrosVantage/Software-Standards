Avoid using Dynamic instantiation

Technology

JEE

Critical Quality Rule

Weight in Parent Tech. Criterion

4

Rationale

Dynamic instantiation is slower than a regular Class invocation or Method call

Description

Dynamic instantiation should be used only when necessary. Dynamic instantiation is done through calls to: - 'java.lang.Class.newInstance()' - 'java.lang.reflect.Constructor.newInstance()' - 'java.lang.Class.getMethod(String name, Class... parameterTypes)' - 'java.lang.Class.getMethods() - 'java.lang.Class.getField(String name)' - 'java.lang.Class.getFields()' - 'java.lang.reflect.Method.invoke(Object obj, Object... args) methods.

Output

This report lists all Artifacts using reflection API methods. It provides the following information: - Artifacts full name - the method of the reflection API' that has been called
