Avoid undocumented Methods

Technology

.Net, ABAP, C#, C++, PowerBuilder, Visual Basic

Unified Status

Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

7

Rationale

Maintainability of the code is facilitated if there is documentation in the code. This rule will ensure there are comments within the Artifact.

Description

Methods should be documented. This reports shows all methods with no comments. Methods include constructors, destructors, and ABAP Event Method. For C#, it excludes getters and setters if the properties/indexer they belong to is documented

Remediation

Add comments into method implementation to explain what is their goal and how they work.

Output

This report lists all methods that are not documented. It provides the following information: Method, Event Method, Constructor, Destrcutor full name

Total

Number of Methods, Event Methods, Constructors and Destructors
