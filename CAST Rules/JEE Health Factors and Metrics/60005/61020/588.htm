Avoid using Global Variables

Technology

C++

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

Global variables should be encapsulated in a class.

Description

This rule searches for C++ code using global variables.

Remediation

Create a static data member in the appropriate class to replace the global variable.

Sample
int m_iGlobal = 0;

void m()
{
 m_iGlobal = 1; // VIOLATION
}
Output

Associated to each C++ Method/Constructor/Destructor with violations, the Quality Rule provides: - the global variable used.
