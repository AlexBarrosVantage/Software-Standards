Good use of Interfaces when instantiating a collection

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

Using the implementation class of a collection as the type of your instance ties your code to this specific implementation and makes it difficult to change the implementation class in the future. For example, if you use the java.util.Collection interface, you can change your implementation class from a Set to a List or to a Queue according to your needs and performance requirements.

Description

Report any fields declaration whose type is a class that implements an interface of type java.lang.Collection, java.lang.Collection<T>, java.lang.Map or java.lang.Map<U,V> or that implements an interface that inherits from these any of the aforementioned interfaces.

Remediation

Declare the type as an interface of a more appropriate collection.

Reference

http://java.sun.com/docs/books/tutorial/collections/interfaces/index.html

Output

This report lists all the fields initialized with the Collection and Map implementation class. It provides the following information: - Field Full Name - Collection Class Fullname

Total

The total number of fields whose type implements or extends java.lang.Collection, java.lang.Collection<T>, java.lang.Map or java.lang.Map<U,V>
