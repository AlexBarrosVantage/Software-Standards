A Copybook should not include other Copybooks

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

For maintenability reasons it is better to avoid defining copybooks including other copybooks. The code becomes difficult to understand and so to maintain.

Description

This rule shows all copybooks including other copybooks.

Remediation

Try to avoid including copybooks in copybooks. The required copybooks should be explicitly included in the calling programs.
