Rationale

For readability aspects, naming conventions must be respected.

Description

Interfaces should always begin with the prefix as defined in the naming convention. This reports shows all the Interfaces not starting with the defined prefix. The prefix is a parameter and can be changed at will. Prefix for ABAP : IF_ Prefix for VB .Net Technology I Prefix for C# Technology I Prefix for C++ Technology I Interfaces in C++ are defined as Classes that have at least one Pure Virtual method. The prefix for C++ is set to I

Remediation

Change name according to the naming convention.

Reference

help.sap.com Naming Conventions in ABAP Objects VB: http://msdn.microsoft.com/en-us/library/0b283bse.aspx

This report lists all Interfaces that does not respect the naming conventions. It provides the following information: Interface full name
