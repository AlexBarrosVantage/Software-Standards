Avoid direct or indirect remote calls inside a loop

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

9

Rationale

All remote calls made inside a loop will badly impact the performances of your application.

Description

Reports the following remote call inside a loop at a depth level less than <depth level>: * SQL Statement through loop (example: SQL cursor on SQL Server, nested cursors on Oracle) * Stored procedure called many times from the client in a loop. * EJB3 Session remote method * ABAP RFC <depth level> is a parameter that can be changed at will.

Remediation

This loop execution can be delegated to the server side of the application so that not network latency will occur.

Output

This report all remote call inside a loop. It provides the following information: - Caller Artifact Full Name that contains the loop - path until the remote called artifact Note that: 1/ The call path may contains different path while only one is an error. For example: g(){myException()} f() { while { g ()} g() } will report both path to g() while only one is a violation. 2/ 2/ Several paths for the same method violating the rule are identified in the Metric Detail page by a sequence ID (in the Value column), this ID is dynamically set during each snapshot and the same path can have a different sequence ID in different snapshots.

Total

All methods calling at least one method inside a loop
