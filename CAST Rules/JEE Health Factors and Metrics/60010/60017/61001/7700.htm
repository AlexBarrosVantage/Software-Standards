Use only Struts HTTP Servlet

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

When Struts framework is used, it must be used for every page in the application to avoid mixing different technologies and creating an application difficult to maintain and be the source of a lack of robustness.

Description

When Struts is used, reports all Servlet that inherit from javax.servlet.http.HttpServlet but not from org.apache.struts.action.ActionServlet class.

Remediation

Adapt the application to use Struts for all HTTP Servlets.

Sample
---> web.xml

<!-- Struts Action Servlet -->
  <servlet>
    <servlet-name>action</servlet-name>
    <servlet-class>
      org.apache.struts.action.ActionServlet
    </servlet-class>
    <init-param>
      <param-name>config</param-name>
      <param-value>/WEB-INF/conf/struts-config.xml</param-value>
    </init-param>
  <servlet>

  <servlet>
    <servlet-name>AnotherServlet</servlet-name>
    <servlet-class>
      // doesn't inherit from org.apache.struts.action.ActionServlet
      // but inherits from javax.servlet.http.HttpServlet
      com.cast.AnotherServlet // VIOLATION:    
   </servlet-class>
  </servlet>
Output

This report lists all HTTP Servlets that don't inherit from Struts action Servlet. It provides the following information: Servlet Full Name, Servlet Class Name

Total

The total is the total number of Servlet that inherit from javax.servlet.http.HttpServlet and it must be 0 when org.apache.struts.action.ActionServlet is not used.
