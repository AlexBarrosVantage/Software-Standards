Avoid large number of String concatenation

Technology

.Net, C#

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

String concatenation resolved at runtime is much slower than using StringBuilder.

Description

All artifacts should not call more than X times the += operator or Concat Method Limitation: This does not apply to C# code contained directly into an ASP.net page (with <% tags)

Remediation

Use StringBuilder and StringBuilder.Append() method instead

Sample
// C# sample
String result = "hello";
result += " John";
result += " Smith"; // VIOLATION if the threshold is 3
Remediation sample
// C# sample
StringBuilder result = new StringBuilder("hello");
result.Append( " John");
result.Append( " Smith");
Output

This report lists all artifacts calling more than X times the '+=' operator or Concat method on String object. It provides the following information: Method full name, the number of time the '+=' or Concat is called
