Avoid Classes with a High Public Data Ratio (PowerBuilder)

Technology

PowerBuilder

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

2

Rationale

Public fields can be accessed by any other Class, therefore their value can be modified at any time, without control by the Class itself. This goes against OO Encapsulation concepts.

Description

Avoid Classes with a High Public Data Ratio > X %. The Public Data Ratio is the percentage of public fields in the total number of all fields. The threshold is a parameter and can be changed at will.

Remediation

Change the field's visibility to private and provide appropriate accessors.
