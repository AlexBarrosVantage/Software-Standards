Rationale

Direct usage of module variables outside of the class prevents control and should be avoided.

Description

Avoid directly accessing module variables from outside the Class not using getters and setters.

Remediation

Define getters and setters even if no checks are in place (such as setting a correct value). If these are needed later on, future modification will be easier.

Output

This report lists all direct accesses to module variables from outside the Class not using getters and setters.
