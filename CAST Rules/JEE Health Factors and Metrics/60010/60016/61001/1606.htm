Triggers should not directly modify tables, a procedure or function should be used instead

Rationale

To optimize the maintenance of the code, modifying data in a table should always be done through a procedure or a function. This enables the centralization of update, insert or delete operations inside appropriate procedure or function making future changes easier to implement.

Description

This metric retrieves all database triggers which directly access tables for update, insert or delete operations.

Remediation

Modify the design of the Trigger so that data modification is done by calling the appropriate procedure or function.

Output

This report lists all Triggers directly modifying data (insert, update or delate) in Tables. It provides the following information: - Trigger full name, - accessed Table

Total

total is the number of schema triggers
