Avoid missing default in switch statements

Technology

C Language, C#, C++, JEE

Unified Status

Unified

N/A

No

8

Rationale

The keyword 'default' is executed when none of the conditions being tested for in the switch statement are met or executed. Having no default keyword, means that there is no backup. The cases that are "impossible" today are those most likely to be the causes of untraceable bugs in the future, when the impossible changes to the standard.

Description

This rule searches for artifacts that contain no 'default' statement in a 'switch' block.

Remediation

Add the 'default' statement with appropriate handling of unexpected values.

Sample
// Sample for C/C++
switch(color)
{
case Red:
    // Handle red case
case Black:
    // Handle black case
} // VIOLATION
Remediation sample
// Sample for C/C++
switch(color)
{
case Red:
    // Handle red case
case Black:
    // Handle black case
default:
    assert(false && "Impossible node color");
    // Error recovery code
}
