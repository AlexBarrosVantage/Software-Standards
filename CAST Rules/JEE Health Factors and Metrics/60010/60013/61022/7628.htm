Avoid large Classes - too many Data Members

Technology

ABAP

Unified Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

For maintainability and readability reasons, classes should not have too many Data Members.

Description

Classes should have less than a defined number of Data Members. The threshold is a parameter and can be changed at will.

Remediation

Review the Class design to reduce the number of data members.

Output

Associated to each ABAP Class with violations, the Quality Rule provides: - The number of class members
