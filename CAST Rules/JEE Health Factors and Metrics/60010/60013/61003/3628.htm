Rationale

Referencing down the inheritance tree is against Object-Oriented practices. It one way to have two-way inheritance between class and subclass. This is an indication of a poor class design and class inheritance. These practices increase the complexity of the code and decrease maintainability.

Description

A Superclass is not allowed to have knowledge of one of its Subclasses. The Superclass has knowledge of the Subclass if the Superclass directly calls a Subclass-method, uses a Subclass-attribute or refers to the name of the Subclass.

Remediation

Review the design of the Class hierarchy.

Output

This report lists all Classes referencing a Subclass. It provides the following information: Class full name, referenced Class full name.
