Avoid Programs with Group By (Cobol)

Technology

Cobol

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

9

Rationale

Having complex queries in Cobol Programs can cause performance problems.

Description

Queries using Group By is considered complex and not performant.

Remediation

Try to avoid using Group By in queries or the number of complex queries.

Output

Associated to each COBOL Program with violations, the Quality Rule provides: - The number of Group By
