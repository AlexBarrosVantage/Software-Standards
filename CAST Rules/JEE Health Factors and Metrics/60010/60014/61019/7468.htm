Disable constraints before merging DataSet

Technology

Unified Status

N/A

Critical Quality Rule

Weight in Parent Tech. Criterion

Rationale

By disabling/enabling constraints before/after call to Merge method, the merge process is quicker.

Description

Call to Dataset.Merge method should be done with disabled constraints.

Remediation

Disable constraints before merging the document and re-enable them after the Merge using the EnforceConstraint property.

Reference

MSDN

Sample
dataset1.Merge( dataset2);
Remediation sample
dataset1.EnforceConstraints = false;
dataset1.Merge( dataset2);
dataset1.EnforceConstraints = true;
Output

List all artifacts where a call to a Merge is done and where EnforceConstraint is not set to false before the method call (in the same method).

Total

Total number of functions calling Merge method.
