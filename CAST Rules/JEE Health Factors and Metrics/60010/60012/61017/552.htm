Rationale

For readability aspects, naming conventions must be respected

Description

This rule searches for C++ methods that do not respect naming conventions. Note: Methods should start with a lowercase character and symbols such as '_' are also to be avoided. Constructors are not taken into account.

Remediation

Change name according to the naming convention
