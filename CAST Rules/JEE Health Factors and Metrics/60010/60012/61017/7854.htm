Database table naming convention - prefix control

Technology

Rationale

For readability aspects, naming conventions must be respected.

Description

For maintainability and readability reasons, names of tables should start with a prefix. This reports shows all the database Tables not starting with the prefix. The prefix value is a parameter and can be changed at will.

Remediation

Change name according to the naming convention.

Reference

Enhancing the Quality of ABAP Development, Meijs-Krouwels-Heulmans-Sommen, SAP Press ISBN 1-59229-030-2

Sample
Sample valid for SAP applications :
The following name does not respect naming conventions for SAP database tables: ZORDER
Remediation sample
Remediation valid for SAP applications :
The name should be: ZTORDER or ZT_ORDER.
Output

This report lists all database tables that does not respect the naming conventions. It provides the following information: Table full name
