Declare as static all Methods not using Instance Fields

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

When an object is created: * Memory is allocated to all the fields * All super class fields are also allocated memory * All sub class fields, super class fields are initialized . * The constructor is invoked. Using a static avoid to create an object that takes resources when it is unnecessary.

Description

All methods that don't use instance fields and/or methods will be reported except if they have been extended in a subclass or inherit from their parents.

Remediation

Make this method static.

Sample
class Example {
    static double pi = 3.14;

    int min (int x, int y) { // VIOLATION
      return (x>y?y:x);
    }

    double getPi() {  // VIOLATION
       return pi;
    }
Remediation sample
class Example {
    static double pi = 3.14;

    static int min (int x, int y) { // FIXED
      return (x>y?y:x);
    }

    static double getPi() {  // FIXED
       return pi;
    }
Output

This report lists all non-static methods that don't use any instance fields. It provides the following information: Method Full Name

Total

The number of non static methods.
