Avoid large Classes - too many Methods

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

For maintainability and readability reasons, Classes should not have too many Methods.

Description

Classes should have less than a defined number of Methods. The threshold is a parameter and can be changed at will.

Remediation

Review the Class design to reduce the number of Methods

Output

Associated to each ABAP Class with violations, the Quality Rule provides: - The number of Methods/Event Methods/Constructors

Total

Number of ABAP Classes
