Avoid Variables declared as Variants

Technology

Visual Basic

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

7

Rationale

Avoid variants when possible. They are slow, and they consume memory. If you use variant instead of, say, integer you waste 14 bytes of memory. This can be significant in a large project. Integers are much faster than variants. This is true particularly in For..Next loops Also if you plan to upgrade to VB.Net, variant are not supported.

Description

Declaration of a variable should not be like : Dim var as Variant

Output

This report lists all declarations of a variable like : Dim var as Variant
