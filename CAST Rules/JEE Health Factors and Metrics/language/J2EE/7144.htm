Action Artifacts should not directly use database objects

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

7

Rationale

Action artifacts should not directly use database objects. All the database access code should be encapsulated behind the business API classes, so Struts doesn't know what persistent layer you are using. This lets you use the same business API classes in other environments, and also to run unit tests against your business API outside of Struts or a HTTP environment.

Description

For all action class (and parent classes) that are referenced in the struts-config.xml through the tag 'action-mapping' (in struts 1.x) or in struts.xml and its included files through the tag 'action' (in struts 2.x), list artifacts that reference database access

Remediation

Use a Business Delegate to perform any business specific operations.

Reference

http://javaboutique.internet.com/tutorials/strictly_struts/index.html

Output

This report lists all Action artifacts having accesses to the database. It provides the following information: - Action artifact full name - database object used

Total

The total number of artifacts of classes and parent classes that are referenced as an action in the XML configuration file.
