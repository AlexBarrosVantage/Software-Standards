Avoid direct Class inheritance from java.lang.Throwable

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

8

Rationale

All Exception Subclasses from java.lang.Throwable will be runtime Exceptions and will not need to be explicitly caught by a try and catch block. This can lead to an uncaught exception.

Description

Classes should not inherit from java.lang.Throwable.

Remediation

Review the design of the Class

Output

This report lists all Static Classes inheriting from java.lang.Throwable. It provides the following information: Class Name, Class Full Name
