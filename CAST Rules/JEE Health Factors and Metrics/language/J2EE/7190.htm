Validator form validate() Method must call super.validate()

Unified Status

Critical Quality Rule

Weight in Parent Tech. Criterion

Rationale

Struts Validator can be extended to implement your own custom validation. When overriding the validate() method, call the ancestor validate() method, to ensure that any framework validations are triggered. In the case the validation framework is disabled, you expose the application to any type of attacks based on invalid field input.

Description

List all validate() methods that override the validate method of the following classes (at any depth in the inheritance tree): org.apache.struts.validator.ValidatorForm org.apache.struts.validator.ValidatorActionForm org.apache.struts.validator.DynaValidatorForm org.apache.struts.validator.DynaValidatorActionForm and that don't call super.validate()

Remediation

Call super.validate() in the validate() method.

Reference

http://cwe.mitre.org/documents/sources/SevenPerniciousKingdoms.pdf

Sample
public ActionErrors validate(
      ActionMapping mapping,
      HttpServletRequest request) {
  if (notGood(mapping,request)) 
    // some code
  }
// some code
}  // VIOLATION
Remediation sample
public ActionErrors validate(
      ActionMapping mapping,
      HttpServletRequest request) {
  ActionErrors errors = super.validate(mapping, request); // FIXED

  if (errors==null) {
     if (notGood(mapping,request)) {
        // some code
     }
  // some code
}
