Avoid static Fields that are not final

Technology

JEE

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

9

Rationale

An enterprise bean must not use read/write static fields. Using read-only static fields is allowed. Therefore, it is recommended that all static fields in the enterprise bean class be declared as final. This rule is required to ensure consistent runtime semantics because while some EJB containers may use a single JVM to execute all enterprise bean's instances, others may distribute the instances across multiple JVMs.

Description

Reports all enterprise beans (session, message and entity) that have not final static fields even if these static fields are hold by a class in its inheritance tree.

Remediation

Declare these static fields as final or review your design.

Reference

JSR-220 Enterprise Java Beans 3.0 Final Release (ejbcore) page 545

Sample
@Stateful
public class MyBean implements MyRemoteBean {
   private static String myName = "MyBean"; // VIOLATION
    ...
    public int aMethod() {
        ...
    }
   ...
}
Remediation sample
@Stateful
public class MyBean implements MyRemoteBean {
   private static final String myName = "MyBean"; // FIXED
    ...
    public int aMethod() {
        ...
    }
   ...
}
Output

This report lists all java enterprise classes that have non final static fields. It provides the following information: - Class full name - list of non final static fields
