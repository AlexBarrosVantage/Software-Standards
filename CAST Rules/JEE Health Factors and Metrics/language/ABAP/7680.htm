Declare as Final all classes that will not be sub-classed

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

There can be situations in which it is necessary to protect a whole class from uncontrolled specialization. This could be suitable, for example, if changes must be made to a class without any subclasses becoming syntactically or semantically incorrect. For a final definition, use the FINAL addition in the CLASS statement.

Description

This rule shows ABAP classes that do not have subclass and that are not defined with the FINAL addition.

Remediation

Check if the class could be inherited. If not, then add the FINAL addition in its definition.
