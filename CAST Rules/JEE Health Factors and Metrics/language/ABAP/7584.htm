Avoid accessing multiple times the same SAP Table or View in an SAP Program (UPDATE)

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

3

Rationale

For maintainability and performance aspect, avoid making access multiple times to the same database table or view in the same programs.

Description

This metric retrieves the ABAP programs making more than a defined number of accesses to the same database table or view. If a program defines classes, then methods belonging to the first level of classes are taken into account. The number of accesses is a parameter of the rule.

Remediation

Try to use a common component to make access to the database table or view. The different parts of the program that need to make access to the same database table or view will have to call this component.

Output

Associated to each ABAP Program with violations, the Quality Rule provides: - The ABAP Table/View accessed multiple times

Total

Number of ABAP Programs
