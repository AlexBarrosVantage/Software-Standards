Avoid Open SQL queries in loops

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

7

Rationale

For performance reason, it is better to avoid accessing database in loops.

Description

The rule searches for SELECT, INSERT, UPDATE, MODIFY, DELETE statements that are contained in loops. These loops are: - SELECT ... ENDSELECT - LOOP ... ENDLOOP - DO ... ENDDO - WHILE ... ENDWHILE - PROVIDE ... ENDPROVIDE

Reference

SAP

Sample
loop at i_incl.
      select single * from trdir into s_trdir
      where name = i_incl-rep
        and cdat in s_cdat
        and udat in s_udat.
    if sy-subrc ne 0.
      continue.
    endif.
endloop
