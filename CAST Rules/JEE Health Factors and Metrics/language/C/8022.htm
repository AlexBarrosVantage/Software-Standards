Avoid hiding attributes

Technology

C++, JEE

Unified Status

Unified

N/A

6

Rationale

Hiding data members is not good Object-Oriented practice. It is a redefinition of an instance or class variable and it makes the relationship between the base class and the derived class unclear. It is also likely to be an unintended mistake.

Description

This rule searches for classes that override data members in the inheritance tree. A data member <member> of class <BaseClass> is overridden in a derived class <DerivedClass> if <DerivedClass> contains an identical declaration of <member> (i.e. same name (type can be different)). A list of data member name exceptions can be provided to manage specific cases where the field is generated or required to be used by a specific framework.

Remediation

The solution depends on the design: - use the inherited member and remove the member declaration in the child class - rename the member of the child class

Sample
Java Sample:
public class SampleBase {
     protected long a = 4;
 }
 
 public class Sample extends SampleBase {
    protected int a = 5;  // VIOLATION
 }

C++ Sample:
class BaseClass 
{
protected: 
    long a;
};
 
class DerivedClass : BaseClass 
{
protected: 
    int a;  // VIOLATION
 };
