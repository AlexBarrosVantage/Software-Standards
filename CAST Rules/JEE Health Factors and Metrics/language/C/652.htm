Avoid Classes with a High Public Data Ratio (C++)

C++

Unified Status

Not Unified

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

2

Rationale

Public fields can be accessed by any other Class, therefore its value can be modified at any time, without control by the Class itself. This is contrary to OO Encapsulation concepts.

Description

Avoid Classes with a High Public Data Ratio > X %. Public Data Ratio is the percentage of public fields in the total number of fields. The threshold is a parameter and can be changed at will.

Remediation

Change the field's visibility to private or make it Const.

Output

This report lists all Classes with a Public Data Ratio greater than X %. It provides the following information: Class full name, PDC
