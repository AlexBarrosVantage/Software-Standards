How to enable the XXL tables Quality Rules using the Table Size Folder option

XXL tables Quality Rules are performance related Quality Rules that help detect incorrect SQL queries running on XXL tables.

XXL tables are extremely large tables, which contain a huge amount of data. The threshold that determines when a table is considered to contain a huge amount of data can be configured by the user (in the CAST Management Studio using the Assessment Model editor - the default is 100000).

In order to enable these XXL tables Quality Rules it is necessary to provide the Snapshot Preparation Assistant in CAST Analysis Manager with the table row size information. This can be done using the Table Size Folder option in the Job Definition page. This folder will then contain XML files defining your table row size information. These XML files must use the extension .SQLTABLESIZE - a syntactically correct example file is displayed here (for Microsoft SQL Server, Oracle Server, Sybase ASE, IBM-DB2, and IBM-zOS) and here (for SAP)

See the Job Definition page in the on-line Help for CAST Analysis Manager for more precise information about this option.

Determining server and database/schema names to use in the .SQLTABLESIZE files

The server, database and schema nodes in the .SQLTABLESIZE file must match the information stored in the CAST Analysis Service following an initial analysis. You can determine this information in several ways:

  • Using CAST Enlighten > use the Property window (right click a table in the Graphical View or in the Technical Browser and select Properties or select the object and press F12) to view the information required, for example for Oracle Server with the legacy PL/SQL Server:


     
  • View table full names in CAST Enlighten> The full names of tables contain the required information. For example for Microsoft SQL Server:

    WESLEY.CASTPUBS..roysched = SERVER.DATABASE..table
  • Using CAST System Views > You can query the CAST System View (CSV) CDT_OBJECTS to extract the information you need, for example:
Select OBJECT_NAME
From CDT_OBJECTS 
Where OBJECT_TYPE_STR = 'SQL Server Database'

Notes

  • Note that when working in an Oracle Server environment please ensure you know whether the schema objects were analyzed with the legacy PL/SQL Analyzer or the SQL Analyzer. The .SQLTABLESIZE file has specific configuration for each, and note that the method of determining the required server/schema names will differ on each.
