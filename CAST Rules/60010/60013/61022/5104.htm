Avoid large Programs - too many Sections

Technology

Cobol

Unified Status

N/A

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

For maintainability aspect, COBOL programs with more than 10 Sections

Description

Avoid Programs with more than X Sections. The threshold is a parameter and can be changed at will.

Remediation

If two sections do the same actions then try to merge them. This can improve the readability, the reusability and maintainability of the code. If it is not possible then try to split the program by extracting its functionalities into subprograms.

Output

Associated to each COBOL Program with violations, the Quality Rule provides: - The number of Sections
