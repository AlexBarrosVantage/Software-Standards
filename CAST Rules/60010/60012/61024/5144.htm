Avoid using GOTO statement

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

7

Rationale

Using GOTO code breaks the process execution flow and makes the code more difficult to understand and maintain.

Description

This rule searches for Cobol programs using GOTO statements to manage the control flow. The "GOTO" statement of embedded SQL is not taken into account.

Remediation

Try to restructure the program and replace GOTO jumps with PERFORM.

Reference

IBM Enterprise Cobol for z/OS - Programming Guide
