Avoid large Interfaces - too many Methods

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

4

Rationale

For maintainability and readability reasons, Interfaces should not have too many Methods.

Description

Interfaces should have less than a defined number of Methods. The threshold is a parameter and can be changed at will.

Remediation

Review the Interface design to reduce the number of Methods.

Output

Associated to each ABAP Interface with violations, the Quality Rule provides: - The number of Methods

Total

Number of ABAP Interfaces
