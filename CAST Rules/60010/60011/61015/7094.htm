Avoid Programs with more than 3 levels of inclusion

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

ABAP programs should not include includes with includes. High levels of inclusion decreases performance.

Description

This rules searches for all ABAP programs using includes which also call includes. The number of inclusions is a parameter (default value = 3).

Remediation

Try to avoid calling includes in includes. The required include files should be explicitly included in the calling programs.

Output

Associated to each ABAP Program with violations, the Quality Rule provides: - The number of inclusion levels
