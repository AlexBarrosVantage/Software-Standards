Avoid Artifacts with too many lines of code (PL/SQL Oracle)

Technology

Forms, PL/SQL

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

6

Rationale

Artifacts with lot of code are more difficult to understand, and are a sign of a bad modularity of the code. Artifacts are functions, stored procedure, triggers and packages

Description

Artifacts should not have more than X lines of code. The threshold is a parameter and can be changed at will.

Remediation

Review the design of the Artifact to reduce its size

Output

This report lists all Procedures, triggers, functions & packages with more than X lines of code It provides the following information: Artifact name, Artifact full name, number of lines of code
