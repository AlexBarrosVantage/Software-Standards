Avoid accessing SAP standard Tables in modification from custom code

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

9

Rationale

SAP standard tables are part of the SAP software and they must not be modified by custom components. These can be affected by the modification of the SAP system tables, in terms of structure, content or business rules. In addition, they can lead to data corruption and unpredictable behavior of SAP packages. The best way is to use standard functions and badis released by SAP.

Description

This rule searches for all custom programs, user-exits, includes, functions, and methods with OpenSQL queries that access SAP standard tables in modification. SAP standard tables are identified as follows: - Their name does not start with Y and Z - They do not belong to a user namespace Custom components are identified as follows: - Their name starts with Y or Z - They belong to a user namespace

Remediation

You should call SAP standard functions and badis to modify the content of SAP standard tables.

Output

Associated to each ABAP custom components with violations, the Quality Rule provides: - The number of violation patterns

Total

Number of ABAP custom Programs, User-exits, Includes, Functions, and Class Methods that contain OpenSQL queries
