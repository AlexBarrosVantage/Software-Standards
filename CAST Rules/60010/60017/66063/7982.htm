Avoid using strlen() function family

Rationale

The 'strlen()' function requires to have a null-terminated sequence of chars to return the size of the string. Otherwise it will return an incorrect length that may lead to memory allocation troubles or security problems.

Description

This rule searches for C/C++ functions, methods, constructors, and destructors that call 'strlen' group of functions.

Remediation

Check if the buffer is correctly null-terminated before using the 'strlen()' function. This can be done by setting the 'sizeof(buf)-1' position to 0. You can also use a more secure variant of the function: 'strlen_s()'.

Reference

Build Security In (https://buildsecurityin.us-cert.gov/bsi/articles/knowledge/coding) http://msdn.microsoft.com/library/default.asp?url=/library/en-us/vclib/html/_crt_strlen.2c_.wcslen.2c_._mbslen.2c_._mbstrlen.asp

Sample
char dest[50];
strncpy(dest, src, strlen(src));
Remediation sample
char dest[50];
strncpy(dest, src, sizeof(dest));
dest[sizeof(dest)-1] = 0;
