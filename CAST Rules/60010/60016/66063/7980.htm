Avoid using strtrns() function

Rationale

The 'strtrns()' function is considered as risky in terms of security because it is possible to overflow the target buffer. If the source buffer is larger than the target buffer, then an overflow will occur.

Description

This rule searches for C/C++ functions, methods, constructors, and destructors that call the 'strtrns()' function.

Remediation

In case of static buffer, insert buffer overflow detection code before calling the function. Otherwise you can use 'malloc()' to create a dynamic buffer adjusted to the input buffer size.

Reference

Build Security In (https://buildsecurityin.us-cert.gov/bsi/articles/knowledge/coding)

Sample
void foo(char *curString)
{
  char newString[20];
  strtrns(curString, oldpattern, newpattern, newString);
}
Remediation sample
void foo(char *curString)
{
  char *newString;
  newString = (char *)malloc(strlen(curString));
  strtrns(curString, oldpattern, newpattern, newString);
}
