Never access an IMS segment with an access mode not authorized in the PCB

Technology

Cobol

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

5

Rationale

The access mode to an IMS segment is defined in the PSB by using the PROCOPT clause of the corresponding PCB. The programs which access to the IMS database must respect this access mode. If not, then errors can occur in production: the programs can abort and the database could be corrupted.

Description

Access to IMS segment must respect the access mode defined in PCB with the PROCOPT clause.

Remediation

Check if IMS segment accesses respect the PROCOPT clause defined for PCB in PSB. If not, then adapt the PROCOPT clause and the accesses made in the Cobol programs.

Output

Associated to each COBOL Program with violations, the Quality Rule provides: - The IMS PCB name

Total

Number of Cobol Programs accessing IMS database (and not running under CICS).
