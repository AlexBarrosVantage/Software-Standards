Avoid Objects having exposed pointers allowed to access unmanaged memory

Technology

C#

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

8

Rationale

For an application, if the pointers are exposed malicious code can change the value of the pointer, potentially allowing access to arbitrary locations in memory or causing application or system failures.

Description

This rule searches all the objects having IntPtr and UIntPtr types as public or protected fields that are used to access unmanaged memory.

Remediation

try and always have these pointers as read-only, internal, or private.

Reference

http://msdn.microsoft.com/en-us/library/ms182306.aspx

Sample
using System;

namespace SecurityRulesLibrary
{

   public class ExposedPointers
   {
      // Violation. The pointers should not be visibles
      public IntPtr Pointer1;
      public UIntPtr Pointer2;
    

}
Remediation sample
using System;

namespace SecurityRulesLibrary
{

   public class ExposedPointers
   {
    
      // Violation Corrected
      internal UIntPtr Pointer1;
      private UIntPtr privatePointer2;

   }
}
Output

This report lists all the classes and structures having IntPtr and UIntPtr types as public or protected fields that are used to access unmanaged memory. It provides following information: - Class name or Structure name
