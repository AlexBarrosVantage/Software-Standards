Avoid nested loops

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

8

Rationale

If the amount of data is large, nested loops are fully avoided due to performance issues. If the program is extracting small amount of data, then focus on SELECT statements than nested loops. The indexed loop and READ statement using binary search are the best methods to avoid performance issues.

Description

This rule checks all ABAP artifacts containing nested loops. These can be implemented with statements: - LOOP ... ENDLOOP - DO ... ENDDO - WHILE ... ENDWHILE - PROVIDE ... ENDPROVIDE Only constructs based on at least 3 levels of nested loops are considered as violating the rule.

Remediation

Evaluate the amount of data processed in the loop and, if necessary, replace nested loop by indexed loop or READ statement with binary search.
