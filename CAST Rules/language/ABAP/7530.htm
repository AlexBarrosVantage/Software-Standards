Avoid "SELECT *" or "SELECT SINGLE *" queries

Rationale

A query that retrieves all columns of a table with a SELECT * or SELECT SINGLE * can potentially be the source of important performance problems: Such performance problems may arise when the execution of the query returns a large result sets (many row with all the columns may then become a huge amount of data to transport over the network). However, there are some exceptions with standard tables such as parameter tables. Also, when using such queries, one cannot control how the columns will be ordered and returned to the client. This can lead to important data inconsistencies and thus stability issues.

Description

This metric retrieves the ABAP artifacts containing queries using the "SELECT *" or "SELECT SINGLE *" statement. It is possible to specify some tables as exceptions by setting them in the parameters of the rule (you can specify complete names, ex: USR02 or generic names by using '%' character, ex: T% or BUT%).

Remediation

Limit the number of field selected by naming those which are expecting only.

Output

Associated to each ABAP Artifact with violations, the Quality Rule provides: - The number of queries containing "SELECT *" or "SELECT SINGLE *"

Total

Number of ABAP Artifacts containing OpenSQL queries.
