Web Dynpro - Avoid changing the program flow

Technology

ABAP

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

9

Rationale

Within Web Dynpro, the program flow must not be changed. You must avoid statements that leave the current session or start other sessions.

Description

This rule checks Web Dynpro source code for statements that can change the program flow such as "LEAVE PROGRAM", "CALL TRANSACTION", "LEAVE TO TRANSACTION" and "SUBMIT".

Remediation

Avoid using the respective technologies.

Reference

SAP

Sample
* Sample 1:
LEAVE PROGRAM.

* Sample 2:
CALL TRANSACTION 'tcode' AND SKIP FIRST SCREEN.

* Sample 3:
LEAVE TO TRANSACTION 'tcode'.

* Sample 4:
SUBMIT myreport.
