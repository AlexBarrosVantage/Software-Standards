Action Classes should have only one public method

1

Rationale

Action Classes should have only 1 public method, this eases struts usage (one function for one action).

Description

Classes that inherit from org.apache.struts.action.Action with more than one public method will be listed. Exception to this rule: - all classes that inherit from org.apache.struts.action.DispatchAction that is specifically designed to group related actions into a single class and have more than one public method to handle each operation. - all Action class that are also Spring Beans: in this case, setter methods are excluded from the violations

Output

This report lists all Action Classes with more than one public method. It provides the following information: - Action Class full name - public method number

Total

All action classes, except those inheriting from dispatchaction
