Avoid using Structures

Technology

C++

Unified Status

N/A

XXL Status

N/A

Critical Quality Rule

No

Weight in Parent Tech. Criterion

3

Rationale

Structures should not be used. Using Structures is against Object Oriented encapsulation principles because it exposes its Members to anyone, without any possible control.

Description

This rule searches for C++ code using structures.

Remediation

Consider creating a Class encapsulating the Structure

Output

Associated to each C++ Method/Constructor/Destructor with violations, the Quality Rule provides: - The Structure used

Total

Number of C++ methods, constructors, and destructors
