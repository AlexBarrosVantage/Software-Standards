Avoid empty catch blocks (JEE)

Technology

JEE

Unified Status

Not Unified

XXL Status

N/A

Critical Quality Rule

Yes

Weight in Parent Tech. Criterion

6

Rationale

An empty catch block defeats the purpose of exceptions. When an exception occurs, nothing happens and the program fails for an unknown reason. The application can then be in an unknown state that will affect subsequent processing. Since the reasons for the issue (the type of the exception and potential embedded message) are ignored, it will require more time to fix.

Description

In a Try and Catch statement, Catch blocks should have code to handle the thrown exception. If they are empty or only contain comments, the Exception will not be handled.

Remediation

The exception must be handles correctly accordingl to its type.
